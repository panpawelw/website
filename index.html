About my projects in general


My programming language of choice is Java 8. Most of my projects are Web Applications based on Spring Framework, both the older version and the newer Spring Boot configurator. I'm also familiar with with Java Servlet technology.

Most of my Web Applications are connected with a MySQL Database, both using simple and plain JDBC interface, but also Hibernate ORM framework or combination of the latter with Spring Data JPA specification.

I'm familiar with Thymeleaf templates, JSP technology, I know the basics of EL language and JSTL expressions.

I have a basic knowledge of JavaFX (8.0) architecture and I have some experience using JavaFX Scene Builder.

In terms of front-end skills - my HLTM, CSS, Bootstrap, plain Javascript and Jquery were enough to create basic user interface to all my projects and then got slightly improved while creating this website.

My Java programming adventure began on Ubuntu and Spring Tool Suite. After several months I've switched my operating system to Windows and later on my IDE to Intellij. It's purely a matter of preferences, I can deal with all of the aforementioned scenarios.

My preferred version control system is Git. I can perform most important operations using console, but nowadays I mostly use Intellij's integration with this platform.

Creating this portfolio was a chance for me to gain some basic experience with deploying Java Web Applications to Amazon Web Services and Google Cloud Platform.

At this particular moment my focus is on learning JUnit testing and applying Test Driven Development principles. I'm also trying to work on my code becoming well structured, with adequate commentary and Javadoc documentation. When I get tired with all this perfectionism I'm reading about design patterns to get a general picture of it and to unwind a bit.

Apart from what's here on this website, you can find several extra little projects on my GitHub account.

Workout Log


My most recent project is a workout log. A general idea behind this application was maximal possible flexibility of data structure. Workout consists of any number of exercises, each of these with any number of sets. In addition to this, user can attach any number of text notes or media files to every workout element. To achieve it I've decided to serialize workout objects to database and media files are stored in the cloud.

Basic calculators like general strength, maximum rep and it's percentages and other (like Wilks score or powerlifting and weightlifting totals in the future) are available right away. After logging in user can access the main logging functionality and will be able to create new workouts and edit or delete old ones. Later on I plan to include administrator panel for users with relevant privileges.

  • it's a Spring Boot application
  • is't been secured with Spring Security using custom login page
  • Spring Cloud module is used to store media files in Amazon S3
  • Spring Mail module is used to activation emails to users who register
  • uses application events and listeners
  • view layer consists of Thymeleaf templates, JS, jQuery and Bootstrap 4

Social Network


This project is a simple social network app. After creating an account and logging in it's possible to view all posts, add your own post or leave comments. There's a user panel available and you can edit user details there. User details are subject to validation both during registration and edition. It's also possible to send and receive private messages to/from other users. Unread private messages are marked as such.

As before - this is a Maven Web App and combination of MySQL, JPA and Hibernate was used as persistence layer.

  • this project is fully annotation - based, so there is no web.xml file

Crime Empire


This is a turn based browser strategy game. You are a gang leader trying to take over all important spots in Whateverville city. You need to manage your resources, which are money, thugs, working ladies and dealers. Each turn you can attack an enemy point transfer resources between your points and the main resource pool and recruit people. During enemy turn your points get attacked and random events occur.

This application is a Maven project created with a colleague as the final coursework for Coders Lab bootcamp. User interface was based to a large extent on another's bootcamp participant's work - GEOS Project by Szymon Mikolajczak, which we adapted to our needs.

  • validator has been used to verify user account data
  • MySQL database communication is implemented according to JPA specification using Hibernate framework

Book Catalog


This web app is a book catalog and made up of JS / jQuery front-end and Java server.

User is able to perform all basic CRUD operations, which are sent to the server via AJAX calls using GET, POST, PUT and DELETE HTML methods. The controller receives these calls and performs relevant database operations using service layer. Responses are sent in JSON format. In this particular case database is mocked by hashlist of Book objects stored in memory but using real database real is just a matter of implementing the relevant interface.

  • JSON format is implemented using Jackson library
  • dependencies are managed using Maven

Programming School


I created this project while learning how to connect Java Web app to MySQL database using JDBC interface. This application is an example of MVC architectural pattern and it's purpose is programming school management with users belonging to user groups submitting solutions to exercies the were given.

The title screen shows last several solutions. User groups can be displayed and public user details and their submitted solutions can be viewed. There's also admin panel where all CRUD operations can be performed on users, groups, exercises and solutions.

  • separate packets have been used for model, DAO and controllers
  • user inteface consists of JSP pages and servlets
  • JSTL expressions and EL language have been used
  • user passwords are hashed with JBCrypt library
