About me...

My interest in IT dates back as long as I can remember. I've done a fair share of programming in my teens - Basic, Turbo Pascal and Assembler, all of it self- taught. Later on my life went in another direction but my passion never dwindled.

I earned my Master's Degree in Psychology from University Of Wrocław. Later on I emigrated to Northern Ireland where I spent best part of my 10 years there as a sessional interpreter in justice system.

Coming back to Poland seems like a perfect excuse to change careers. Intepreting was nicely profitable and a great choice at the time, but I never had real interest in the profession itself.

After realizing the demand in IT industry is very high, I've decided to take a Coders Lab Back-end bootcamp (Java Web). I took some time to sort my things out, but I kept programming as much as I could to keep my newly learned skills and improve. Eventually, I'd like to become a full - stack developer.

About my projects in general


My programming language of choice is Java 8. Most of my projects are Web Applications based on Spring Framework, both the older version and the newer Spring Boot configurator. I'm also familiar with with Java Servlet technology.

Most of my Web Applications are connected with a MySQL Database, both using simple and plain JDBC interface, but also Hibernate ORM framework or combination of the latter with Spring Data JPA specification.

I'm familiar with Thymeleaf templates, JSP technology, I know the basics of EL language and JSTL expressions.

I have a basic knowledge of JavaFX (8.0) architecture and I have some experience using JavaFX Scene Builder.

In terms of front-end skills - my HLTM, CSS, Bootstrap, plain Javascript and Jquery were enough to create basic user interface to all my projects and then got slightly improved while creating this website.

My Java programming adventure began on Ubuntu and Spring Tool Suite. After several months I've switched my operating system to Windows and later on my IDE to Intellij. It's purely a matter of preferences, I can deal with all of the aforementioned scenarios.

My preferred version control system is Git. I can perform most important operations using console, but nowadays I mostly use Intellij's integration with this platform.

Creating this portfolio was a chance for me to gain some basic experience with deploying Java Web Applications to Amazon Web Services and Google Cloud Platform.

At this particular moment my focus is on learning JUnit testing and applying Test Driven Development principles. I'm also trying to work on my code becoming well structured, with adequate commentary and Javadoc documentation. When I get tired with all this perfectionism I'm reading about design patterns to get a general picture of it and to unwind a bit.

Apart from what's here on this website, you can find several extra little projects on my GitHub account.

Workout Log


My most recent project is a workout log Spring Boot Application. The main goal is maximal possible flexibility of data structure. If workouts of the sports you practice can be broken down to actions of some difficulty performed number of times in (optionally) certain period of time - they should be possible to record in my application. Any training that's aimed at increasing some type of measurable performance (as opposed to the type that relies on style) should be possible to capture in such data structure.

Basic calculators are available without logging in - one repetition max, percentages, powerlifting and weightlifting totals, maybe also Wilks score. After logging in user will be able to access workout log with two main functionalities - viewing or editing past workouts and adding a new one. I'd like to include adding all sorts of notes, voice notes, pictures, photos and videos.

If user has administrator privileges as well, admin panel will be available.

  • this application is secured using Spring Security
  • login and logout pages are custom made
  • JBCrypt library is used as the password encoder
  • Spring Data JPA implemented via Hibernate framework is responsible for database communication
  • Thymeleaf templates and controllers are responsible for view layer
  • application is able to send registration email to an email provided by user

Social Network


This project is a simple social network app. You can create an account and after logging in possible to view posts of all other users, add your own post or leave a comment to any post. It's also possible to send and receive private messages to/from other users. Unread private messages are marked as such.

I wanted to try something different this time and this project is fully annotation - based. There is no web.xml file.

  • database communication is according to JPA specification via Hibernate framework
  • validator is used to verify user account data
  • user passwords are hashed using JBCrypt library
  • dependencies are managed using Maven
  • view layer consists of controllers and JSP files with JSTL expressions and EL language

Crime Empire


This project was the final coursework for Coders Lab bootcamp written together with a colleague. It's a turn based browser strategy game. The player is an aspiring crime boss of Whateverville city and the main goal is to take control over all of the points on the city map.

User data and the game state related to his account are stored in MySQL database using JPA specification and Hibernate framework.

Game's user interface is based to a large extent on the GEOS project by another Coders Lab bootcamp participant - Szymon Mikołajczak.

Managing your resources is crucial:

  • money - everybody has his price...
  • thugs - conquer enemy points on the map and defend yours
  • working ladies and dealers - earn money for you
  • validator is been used to verify user account data
  • user passwords are hashed using JBCrypt library
  • dependencies are managed using Maven
  • view layer consists of controllers and JSP files with JSTL expressions and EL language

Book Catalog


This project is a book catalog and consists of two complementary that complement each other: jQuery front-end sending AJAX calls to Java server that performs database operations.

User interface sends a GET call to the server and receives the list of books that is displayed on the webpage. The user can display every book's details (GET call), edit them (PUT), delete this book from the list (DELETE) or add a new one (POST). Book data is sent and received using JSON format.

Back-end of this project is a Spring Web Application implementing CRUD model. The controller receives AJAX request and then performs database operations via service layer. Mock-up database is a hashlist of Book objects stored in memory, but since interface has been used - implementing real database should be a simple matter.

  • JSON format is implemented using Jackson library
  • dependencies are managed using Maven

Programming School


This project was completed as a part of the boot camp, to learn how to connect Java Web app to MySQL database using JDBC interface. This application is an example of MVC architectural pattern. It simulates programming school management with users belonging to user groups being given certain exercises they can submit solutions to.

The title screen shows last several solutions. User groups can be displayed and public user details and their submitted solutions can be viewed.

Admin panel is also available. Everything can be viewed, added, edited or deleted: users, groups, exercies and solutions.

  • separate classes have been used for model, DAO and controllers
  • user inteface consists of JSP pages and controllers
  • JSTL expressions and EL language have been used
  • database connection is achieved via MySQL Connector/J library
  • user passwords are hashed with JBCrypt library
