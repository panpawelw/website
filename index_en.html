Ogólnie o moich projektach


Piszę w Javie 8, głównie projekty Web przy pomocy frameworku Spring, zarówno w starszej jego wersji, jak i z użyciem konfiguratora Spring Boot. Umiem również posługiwać się technologią serwletów.

W kilku moich projektach użyłem komunikacji z bazą danych MySQL, zarówno za pośrednictwem interfejsu JDBC jak i bibliotek mapowania obiektowo-relacyjnego Hibernate i specyfikacji JPA.

Potrafię posługiwać się systemem szablonów Thymeleaf, technologią JSP, umiem stosować język EL oraz tagi JSTL.

Mam opanowaną w podstawowym stopniu architekturę JavaFX (w wersji 8.0) i umiem posługiwać się JavaFX Scene Builderem.

Jeśli chodzi o front-end - HTML, CSS, Bootstrap, Javascript i JQuery (włącznie z zapytaniami AJAX) mam opanowane na tyle, aby być w stanie stworzyć tą stronę oraz podstawowy interfejs użytkownika do wszystkich moich projektów.

Zacząłem moją przygodę z Javą na Ubuntu i Spring Tool Suite. Po kilku miesiącach przesiadłem się na Windows, a po kolejnych kilku zmieniłem IDE na Intellij. Jest to czysto kwestia preferencji, jestem w stanie poradzić sobie we wszystkich wymienionych scenariuszach.

Jako system kontroli wersji wybrałem GIT. Potrafię wykonać podstawowe operacje z poziomu konsoli, obecnie częściej korzystam z wbudowanej w Intellij integracji z tą platformą, ewentualnie używam nakładki GitHub Desktop.

Przy okazji tworzenia tego portfolio zdobyłem podstawowe doświadczenie w korzystaniu z Amazon Web Services oraz Google Cloud Platform.

Aktualnie koncentruję się na opanowaniu pisania testów przy użyciu JUnit, umiejętności pisania klarownego i dobrze ustrukturalizowanego kodu ze stosownymi komentarzami i dokumentacją oraz rozszerzeniu mojej wiedzy odnośnie wzorców projektowych.

Na moim GitHubie można jeszcze znaleźć kilka drobnych projektów, poza tymi które przedstawiłem poniżej.

Workout Log


Obecnie pracuję nad aplikacją Spring Boot, która ma służyć jako dziennik treningów. Celem jest duża elastyczność struktury danych, dzięki czemu w prosty sposób można dostosować ją do różnych dyscyplin, jeśli tylko trening opiera się o serie, powtórzenia czy elementy wykonywane na czas. Chcę pojedynczy trening traktować jako obiekt, który będzie później serializowany do bazy danych. Osobiście używam tej aplikacji do logowania sesji dwubojowych i trójbojowych.

Bez logowania dostępny ma być ekran podstawowych kalkulacji - maksymalnego powtórzenia wedle różnych dostępnych wzorówi oraz procentów maksymalnego powtórzenia (może też przelicznik współczynnika Wilksa i trójbowy total). Po zalogowaniu użytkownik uzyska dostęp do pełnej funkcjonalności - dziennika treningowego z możliwością wglądu i edycji dawnych treningów oraz dodania nowego treningu. Chcę aby była, poza logowaniem samych danych trenigowych, możliwość dodawania wszelkiego rodzaju uwag, notatek, notatek głosowych, zdjęć i filmów.

Jeśli użytkownik ma również uprawnienia administratora, będzie miał także dostęp do panelu administracyjnego.

  • aplikacja jest zabezpieczona przy użyciu Spring Security
  • jako password encoder zastosowałem bibliotekę JBCrypt
  • użyłem własnych stron logowania
  • za komunikację z bazą danych odpowiada Spring Data JPA zaimplementowane poprzez framework Hibernate
  • jako warstwy widoku użyłem szablonów Thymeleaf
  • aplikacja jest w stanie wysłać wiadomość z potwierdzeniem rejestracji na podany adres email

Social Network

Użytkownik może utworzyć konto, po zalogowaniu do którego ma możliwość obejrzenia wszystkich wysłanych przez innych użytkowników komunikatów, dodania swojego komunikatu, dodania komentarza do swojego lub cudzego komunikatu.   • zależnościami zarządza Maven
  • za warstwę widoku odpowiadaja kontolery oraz technologia JSP z JSTL i EL

Crime Empire


This project was the final coursework for Coders Lab bootcamp written together with a colleague. It's a turn based browser strategy game. The player is an aspiring crime boss of Whateverville city and the main goal is to take control over all of the points on the city map.

User data and the game state related to his account are stored in MySQL database using JPA specification and Hibernate framework.

Game's user interface is based to a large extent on the GEOS project by another Coders Lab bootcamp participant - Szymon Mikołajczak.

Managing your resources is crucial:

  • money - everybody has his price...
  • thugs - conquer enemy points on the map and defend yours
  • working ladies and dealers - earn money for you
  • validator is been used to verify user account data
  • user passwords are hashed using JBCrypt library
  • dependencies are managed using Maven
  • view layer consists of controllers and JSP files with JSTL expressions and EL language

Book Catalog


This project is a book catalog and consists of two complementary that complement each other: jQuery front-end sending AJAX calls to Java server that performs database operations.

User interface sends a GET call to the server and receives the list of books that is displayed on the webpage. The user can display every book's details (GET call), edit them (PUT), delete this book from the list (DELETE) or add a new one (POST). Book data is sent and received using JSON format.

Back-end of this project is a Spring Web Application implementing CRUD model. The controller receives AJAX request and then performs database operations via service layer. Mock-up database is a hashlist of Book objects stored in memory, but since interface has been used - implementing real database should be a simple matter.

  • JSON format is implemented using Jackson library
  • dependencies are managed using Maven

Programming School


This project was completed as a part of the boot camp, to learn how to connect Java Web app to MySQL database using JDBC interface. This application is an example of MVC architectural pattern. It simulates programming school management with users belonging to user groups being given certain exercises they can submit solutions to.

The title screen shows last several solutions. User groups can be displayed and public user details and their submitted solutions can be viewed.

Admin panel is also available. Everything can be viewed, added, edited or deleted: users, groups, exercies and solutions.

  • separate classes have been used for model, DAO and controllers
  • user inteface consists of JSP pages and controllers
  • JSTL expressions and EL language have been used
  • database connection is achieved via MySQL Connector/J library
  • user passwords are hashed using JBCrypt library
